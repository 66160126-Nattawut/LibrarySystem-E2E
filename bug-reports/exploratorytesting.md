# รายงานผลการทดสอบแบบเจาะลึก (Exploratory Testing แบบ SBTM & Heuristics)

**ชื่อโครงการ:** ระบบจัดการห้องสมุด (Library Management System)  
**รูปแบบการทดสอบ:** ท่องระบบแบบเจาะลึกอิงตามช่วงเวลา (Session-Based Test Management)  
**ผู้ทำการทดสอบ:** ผู้ทดสอบระบบ (Manual Tester)  
**วันที่ดำเนินการ:** 2026-04-04

---

## 1. บทสรุปโครงร่าง (Executive Summary)

การทดสอบแบบ **เจาะลึก (Exploratory Testing)** เป็นการทดสอบที่ไม่มีการเขียนสคริปต์ล่วงหน้าตายตัว แต่เน้นให้ผู้ทดสอบใช้ประสบการณ์ ความคิดสร้างสรรค์ และเทคนิคการคาดเดาจุดอ่อน (Heuristics) ออกเดินทางสำรวจ (Test Tours) เพื่องัดแงะระบบในมุมมุมหลืบที่เอกสาร Test Case ปกติครอบคลุมไม่ถึง

เพื่อให้การทำงานมีแบบแผน เราได้ใช้ศาสตร์การแบ่งเวลาแบบ **Session-Based Test Management (SBTM)** เข้ามาตีกรอบการทำงาน โดยจัดแบ่งการพยายามทำลายระบบออกเป็น 4 กิจกรรมหลัก (4 เซสชัน) ดังต่อไปนี้

---

## 2. ผลการเจาะลึกแบ่งตามเซสชัน (Exploratory Test Sessions)

### เซสชันที่ 1: "ทัวร์เส้นทางเงินทอง" และ "ทัวร์โซนอันตราย" (The Money & Bad Neighborhood Tour)

**ภารกิจหลัก:** สำรวจเป้าหมายที่เกี่ยวข้องกับรายได้ของห้องสมุด (เช่น ค่าปรับ) และโซนอันตรายที่มักมีปัญหาบ่อย เพื่อทดสอบตรรกะทางธุรกิจให้พัง  
**กรอบเวลา:** 60 นาที  
**เทคนิคการทดสอบ:** ทดสอบค่าขอบเขตข้อมูล (Boundary Testing), โจมตีข้อจำกัดช่องกรอก (Input Constraint Attack), แอบแปลงค่าพารามิเตอร์ (Parameter Tampering)

**บันทึกข้อค้นพบจากการโจมตี:**

- พยายามข้ามเวลาในระบบ (Time Manipulation) โดยนำหนังสือไปคืนหลังเที่ยงคืนเป๊ะ **[ผลลัพธ์]** พบ **BUG-007**: ระบบข้อมูลใช้โซนเวลามาตรฐาน (UTC) ถือว่าขึ้นวันใหม่และปรับเงินทันที 5 บาท
- ดองหนังสือไว้ 2 ปีแล้วนำมาคืนเพื่อทดสอบเพดานตัวเลข **[ผลลัพธ์]** พบ **BUG-002**: ระบบไม่มีการปิดหลังคาตายตัว (Max Limit) คิดเงินค่าปรับลูกค้าทะลุเป็นพันบาท แทนที่จะล็อกเพดานไว้แค่ 200 บาทตามเอกสารข้อกำหนด
- เข้าหน้าจอชำระค่าปรับ แต่ทดลองกรอกช่องรับเงินเป็น `-500` บาท **[ผลลัพธ์]** พบ **BUG-005**: เว็บหน้าบ้านไฟเขียวให้ชำระจำนวนเงินติดลบ กลายเป็นระบบติดหนี้สมาชิกแทน!

---

### เซสชันที่ 2: "ทัวร์คนย้ำคิดย้ำทำ" และ "การแข่งแย่งชิงทรัพยากร" (The OCD Tour & Concurrency Race)

**ภารกิจหลัก:** กดย้ำๆ คลุ้มคลั่งเหมือนคนเป็นย้ำคิดย้ำทำ (OCD) รวมถึงจำลองการแย่งกันทำรายการ เพื่อค้นหาปัญหาด้านจำนวนทรัพยากรในคลัง  
**กรอบเวลา:** 60 นาที  
**เทคนิคการทดสอบ:** ดึงจังหวะแย่งโหลดข้อมูล (Concurrency Check), ชิงจังหวะเซิร์ฟเวอร์ (Race Condition), กดรัวๆ ไม่ให้พักเรนเดอร์

** บันทึกข้อค้นพบจากการโจมตี:**

- ล็อกอินด้วยสิทธิ์นักเรียน แล้วกดยืมหนังสือติดๆ กัน 10 ทีโดยไม่รอแอนิเมชันโหลดเว็บ **[ผลลัพธ์]** พบ **BUG-001**: ระบบเผลอแจกหนังสือนักเรียนทะลุลิมิตไปถึงเล่มที่ 4 (โควต้าปกติคือ 3 เล่ม)
- เปิด 2 หน้าต่างเบราว์เซอร์ สไตรค์แข่งกันกดยืมหนังสือที่เหลือเพียงชิ้นเดียว (ว่าง 1 เล่ม) ในเสี้ยววินาทีเดียวกัน **[ผลลัพธ์]** พบ **BUG-004**: แจกหนังสือให้ทั้งคู่สำเร็จ สต็อกหนังสือติดลบหลอนกลายเป็น `-1` ทันที (ฐานข้อมูลไม่มีระบบล็อกข้อมูลขณะบันทึก)
- พฤติกรรมประหลาด: พยายามแกล้งกดคีย์ลัดสร้างหนังสือที่ใช้เลข ISBN ซ้ำกันรัวๆ **[ผลลัพธ์]** พบ **BUG-008**: หน้าเว็บขาวโพลน พ่นหน้าแจ้งพัง 500 ทางเทคนิค แทนที่จะเด้งกล่องข้อความเตือนซ้ำแบบนุ่มนวล

### เซสชันที่ 4: "ทัวร์ตามติดพัสดุ (เกาะท่อข้อมูล)" และ "ขีดจำกัดหน้าจอ" (Data Flow & Platform Limitations)

**ภารกิจหลัก:** ตามไล่สังเกตเส้นทางเดินของข้อมูลตั้งแต่ตอนกรอก ยันตอนออกเอกสารรายงาน พ่วงด้วยการจับระบบไปรันบนหน้าจอเน่าๆ  
**กรอบเวลา:** 45 นาที  
**เทคนิคการทดสอบ:** กดดันแพลตฟอร์ม (Platform Stress), เช็คการอ่านตัวอักษร, อัดภาระข้อมูลเรื่อยๆ

** บันทึกข้อค้นพบจากการโจมตี:**

- เข้าไปเปิดหน้าจอสถิติรวม (Dashboard) ด้วยแท็บเล็ตรุ่นเก่าที่มีความละเอียดต่ำ **[ผลลัพธ์]** พบ **BUG-017**: กราฟสถิติหน้าตาอ้วนขยายชนเบียดกันจนดูไม่รู้เรื่อง
- ทดลองกดย้ำๆ 3 ครั้งติดกันที่ปุ่มบันทึกข้อมูลเพราะคิดว่าคอมมือถือค้าง **[ผลลัพธ์]** พบ **BUG-019**: ปุ่มบันทึกไม่มีสถานะหมุนรอ (Loading Spinner) ป้องกันขัดจังหวะ ทำให้ระบบเบิ้ลรับส่งเอกสารซ้ำซ้อน
- ลากเมาส์พยายามเปิดดูคำอธิบายไอคอนในตารางรายการหนังสือ **[ผลลัพธ์]** พบ **BUG-013**: กล่องข้อความลอยตัว (Tooltip) จมมิดหายไปอยู่ใต้ตาราง (ลอจิกชั้นแสดงผล Z-Index ไม่ถูกต้อง)
- เปลี่ยนจอมอนิเตอร์พกพาเป็นมือถือรุ่นเก่าแล้วปรับเข้าโหมดจอภาพสีมืด (Dark Theme) **[ผลลัพธ์]** พบ **BUG-012**, **BUG-014** และ **BUG-020**: ตารางหลุดขอบ จอพินาศ และปุ่มพื้นเป็นสีดำกลืนไปกับตัวอักษร มองไม่เห็นอะไรเลย

## 3. สรุปตารางรวมเทคนิคการเจาะประเด็น (SFDIPOT Matrix)

การลงสนาม Exploratory Testing ครั้งนี้ คุมหางเสือการวิเคราะห์ด้วยสูตรท่องจำยอดฮิต **SFDIPOT** เพื่อให้แน่ใจว่าเราโจมตีครบทุกมิติโครงสร้าง:

| ตัวอักษรทริค (Heuristic)              | เป้าหมายโฟกัส (Focus Point)                     | ผลจากการแหย่ระบบ (Action / Found Issue)                                                                                                   |
| ------------------------------------- | ----------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| **S** - Structure (โครงสร้าง)         | โครงสร้าง URL / เส้นทางการเคาะลิงก์             | พบ **BUG-006** - นักเรียนหรือสมาชิกธรรมดาสามารถมุดผ่านไปโผล่หน้าแอดมินได้สำเร็จ                                                           |
| **F** - Function (ฟังก์ชัน)           | ตรรกะเชิงซ้อนที่เกี่ยวกับการคำนวณตัวเลขและเวลา  | พบ **BUG-007, BUG-002** - คำนวณวันข้ามโซนเวลาผิด และคิดค่าปรับโหดไม่มีการบังคับชนเพดาน                                                    |
| **D** - Data (ข้อมูล)                 | ป้อนข้อมูลประเภทคาบเส้น / ค่าว่าง / ข้อมูลติดลบ | พบ **BUG-005** - ตัวช่องกรอกค่าปรับ สามารถรับเครื่องหมายติดลบ ส่งผลให้บัญชีรายได้ของระบบเพี้ยนรวน                                         |
| **I** - Interface (อินเตอร์เฟซหน้าจอ) | กล่อง input / การแสดงผลตอบโต้ต่างๆ บนจอ         | พบ **BUG-011, BUG-018, BUG-013, BUG-019** - เกิดช่องโหว่ความปลอดภัย, พิมพ์ผิดค้างไม่แจ้ง Error, กล่องเครื่องมือจม, และกดย้ำบันทึกเบิ้ลได้ |
| **P** - Platform (แพลตฟอร์มเครื่อง)   | สภาพแวดล้อมโทรศัพท์มือถือ, ความแคบเบราว์เซอร์   | พบ **BUG-012, BUG-017** - ตารางข้อมูลและกราฟ หลุดแตกกระจายบนการแสดงผลหน้าจอมือถืออย่างหนัก                                                |
| **O** - Operations (ปฏิบัติการร่วม)   | การใช้งานรุมกันพร้อมๆ กัน (Concurrency)         | พบ **BUG-004** - ข้อมูลตีกันตาย ยูนิตของหนังสือเพี้ยนติดลบจากการแย่งกันยืมหนังสือ                                                         |
| **T** - Time (จังหวะเวลา)             | การตั้งค่าที่ผูกติดอยู่กับห้วงเวลาหรือลิมิตอายุ | พบ **BUG-016** - อายุของล็อกอินเซสชัน สั้นกุดจนผู้ใช้งานโดนเตะออกจากระบบเร็วกว่าปกติจนก่อความรำคาญ                                        |

## 4. สรุปปิดท้ายภารกิจสำรวจ (SBTM Debrief)

จากการปล่อยให้ผู้เชี่ยวชาญเข้าผจญภัยโจมตีตามซอกมุมของ Library Management System นอกกรอบ Test Cases ปกติ พบว่าระบบสร้างการครอบคลุม **สถานการณ์ไหลลื่น (Happy Path)** ไว้ได้อย่างสวยงามและนุ่มนวล

แต่เมื่อเจาะลึกลงไปในส่วนของ **สถานการณ์ขรุขระ (Negative Paths)** และ **ข้อมูลจำเพาะสุดขอบ (Edge Cases)** ระบบมี **ระดับความเสี่ยงทางโครงสร้างโค้ดที่ค่อนข้างสูง**
โดยเฉพาะปัญหาร้ายแรงอย่างการที่ระบบฐานข้อมูลไม่ล็อกจังหวะขณะยืม (BUG-004), การหละหลวมของท่อตรวจสอบฝั่งเซิร์ฟเวอร์ต่อตัวเลขติดลบ (BUG-005) และการเพิกเฉยทิ้งช่องโหว่ความปลอดภัยเข้าหลังบ้านตรงๆ (BUG-006)
